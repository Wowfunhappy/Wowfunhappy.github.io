An HTTPS Proxy installer for old versions of Mac OS X

I happen to love older versions of Mac OS X. Mavericks is, in my humble opinion, still the best operating system in existence, and Snow Leopard and Tiger also have their charms.

But one place where older Mac operating systems fall short nowadays is in their HTTPS support. For example, since fall 2020, any Mavericks user who uses the Dictionary app to search Wikipedia will be told the server is unavailable. Wikipedia isn't down, of course, but it no longer supports any of the HTTPS ciphers which Mavericks can understand.

To fix this problem, you need a proxy server—a "man in the middle" which intercepts and fixes traffic before sending it on its way. I decided to go with Squid, because it's lightweight, open source, battle-tested, and Mac-compatible. Use the below installer to set up a pre-configured version on your own PPC or Intel Mac running OS X 10.5 Leopard or newer. The proxy will run locally, so no unencrypted data should ever leave your machine.

Issues this proxy is known to fix include:

  • Searching Wikipedia in the Dictionary app.
  • Loading remote images in Apple Mail.
  • Syncing with Junecloud in Delivery Status.
  • Adding feeds in NetNewsWire.
  • Connecting to GitLab in Tower.
  • Playing live streams in QuickTime.

...and much more!

Download Jonathan’s Mac HTTPS Proxy

Please do not use this proxy to browse the web in ancient insecure versions of Safari. Use Chromium Legacy, InterWeb, or TenFourFox instead.

I have done my best to configure this proxy in a secure manner. It uses a root certificate generated locally at install time to be unique to your machine, and only allows connections from localhost by default. It is my sincere belief that this proxy will increase the security of old machines by giving them access to newer ciphers and protocols—but I am not a network security professional, and I cannot commit to updating this package in a timely manner. Use at your own risk.
