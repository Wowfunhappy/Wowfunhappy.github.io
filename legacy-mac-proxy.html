Legacy Mac Proxy

Make HTTPS work again on old versions of Mac OS X

I happen to love older versions of Mac OS X, but one place where they fall short nowadays is HTTPS support. On OS X 10.9, the Dictionary app is no longer able to connect to Wikipedia, and Apple Mail is filled with broken remote images. To fix these problems, you need a proxy server, and specifically a "man in the middle" proxy which can translate between modern HTTPS protocols and older TLS implementations.

The Squid proxy is a lightweight, open source, battle-tested, and Mac-compatible option. It's a bit tricky to set up, but I've created a pre-configured installer to streamline the process. It will work on any Intel-based Mac running OS X 10.6 or newer, and includes experimental support for PowerPC.

Problems this proxy is known to fix include:

  • Searching Wikipedia in the Dictionary app.
  • Loading remote images in Apple Mail.
  • Syncing with Junecloud in Delivery Status.
  • Refreshing feeds in NetNewsWire.
  • Connecting to GitLab in Tower.
  • Playing live streams in QuickTime.

...and much more! If you're using a version of OS X that's more than seven years old, you should definitely give it a try!

Download Legacy Mac Proxy

Please don't use this proxy to browse the web in ancient insecure versions of Safari. Instead, use Chromium Legacy on OS X 10.7 or above, InterWeb on 10.6, or TenFourFox on 10.5 or 10.4.
